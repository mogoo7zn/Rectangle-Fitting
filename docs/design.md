# 算法设计详细说明

## 1. 问题定义

Rectangle Fitting（矩形装箱）：给定一个尺寸为 W×H 的容器矩形，以及 n 个小矩形 R₁,...,Rₙ（每个原始尺寸为 wᵢ×hᵢ），判断是否存在一种放置方案，使得：
1. 每个小矩形完全在容器内部（不超出边界）
2. 小矩形之间互不重叠
3. 每个小矩形可以选择是否旋转 90°

## 2. Z3 求解器设计

### 2.1 SMT 建模

对每个矩形 i 引入整数变量：
- `x_i`: 左下角 x 坐标
- `y_i`: 左下角 y 坐标  
- `w_i`: 实际放置宽度
- `h_i`: 实际放置高度

布尔变量：
- `r_i`: 是否旋转

#### 旋转约束

若允许旋转（`allow_rotation = true`）：
```
(w_i = wᵢ ∧ h_i = hᵢ ∧ ¬r_i) ∨ (w_i = hᵢ ∧ h_i = wᵢ ∧ r_i)
```

若不允许旋转：
```
w_i = wᵢ ∧ h_i = hᵢ ∧ ¬r_i
```

#### 边界约束

```
x_i ≥ 0
y_i ≥ 0
x_i + w_i ≤ W
y_i + h_i ≤ H
```

#### 非重叠约束（互补情况分裂）

对任意 i < j：
```
(x_j ≥ x_i + w_i) ∨ (x_i ≥ x_j + w_j) ∨
(y_j ≥ y_i + h_i) ∨ (y_i ≥ y_j + h_j)
```

这等价于"两个矩形在 x 方向或 y 方向上分离"，通过德摩根律将 OR 展开。

### 2.2 可选优化

**对称性破缺 (Symmetry Breaking)**：
- 将面积最大的矩形固定在 (0, 0)
- 对同尺寸矩形施加字典序约束（避免排列组合产生等价解）

**面积快速检查**：
- 若 Σ(wᵢ·hᵢ) > W·H，直接返回 unsat

### 2.3 为什么 Z3 适合

- Rectangle Fitting 的约束本质是线性整数约束 + 布尔变量组合 → 天然适合 SMT
- Z3 的 Nelson-Oppen 理论组合能高效处理 Int + Bool 的混合约束
- Z3 内置的 DPLL(T) 框架对这类组合问题有良好支持

## 3. 自定义算法设计

### 3.1 整体框架

借鉴 SAT 求解中 CDCL + Local Search 的混合思路，实现 Hybrid Branch-and-Bound Rectangle Fitter：

```
精确回溯搜索（主体）
    ↓ 长时间无进展时触发
局部改进（辅助加速）
    ↓ 失败则返回主搜索继续回溯
```

### 3.2 状态表示

搜索状态包含：
- `placed`: 已放置矩形列表（含位置、朝向）
- `unplaced`: 未放置矩形列表
- `anchors`: 当前可用候选放置点集合
- 统计计数器

### 3.3 候选点机制

不使用全平面枚举，而是维护离散候选点集：

**初始化**：`anchors = {(0, 0)}`

**更新规则**：放置矩形 i 后，在其右边缘和上边缘生成新候选点：
- `anchor' = (x_i + w_i, y_i)`
- `anchor'' = (x_i, y_i + h_i)`

**剪枝**：去除容器外部的候选点，去除重复候选点。

此机制将搜索空间从 O(W×H·n) 降到 O(n²) 级别。

### 3.4 分支顺序启发式

每次递归优先处理"最难放"的矩形：

1. **面积降序 (AREA_DESC)**：按面积从大到小排序 — 贪心优先大矩形，减少后期空间碎片
2. **最大边降序 (MAX_DIM_DESC)**：按 max(w, h) 降序 — 优先处理最"长"的矩形
3. **最少候选点优先 (FEWEST_CANDIDATES)**：动态计算每个未放矩形在当前状态下有多少个候选点可放，选择最少的 — 聚焦瓶颈

### 3.5 剪枝策略

| 剪枝 | 条件 | 说明 |
|------|------|------|
| 面积剪枝 | 剩余矩形总面积 > 剩余可用面积 | 物理上不可能 |
| 不可放剪枝 | 存在未放矩形在所有候选点都不能放 | 提前失败 |
| 对称性剪枝 | 首个最大矩形只尝试 (0,0) 候选点 | 避免等价排列 |
| 支配剪枝 | 候选点在几何上等价或被支配 | 减少冗余搜索 |

### 3.6 局部改进机制

当精确搜索在某一深度长时间无进展（500ms 无新进展）时，触发局部搜索：

1. **贪心补全**：从当前状态出发，随机打乱未放矩形顺序，贪心尝试所有候选点
2. **扰动修复**：若贪心失败，尝试扰动（移除1~3个已放矩形，重新选择放置点）
3. **最多 20 次尝试**：每次尝试独立随机打乱

局部搜索找到解 → 立即返回 SAT  
局部搜索失败 → 返回主搜索继续回溯

**正确性保证**：局部搜索是加速器，不替代完整搜索。搜索框架仍然是完备的（若存在解则必能找到）。

### 3.7 统计信息

算法记录以下统计量用于分析：
- `recursive_calls`: 递归调用次数
- `backtracks`: 回溯次数
- `placements_tried`: 尝试放置次数
- `pruned_by_area/infeasible/symmetry`: 各剪枝被触发的次数
- `local_search_calls/successes`: 局部搜索调用/成功次数
- `max_depth`: 最大搜索深度

## 4. 正确性说明

### Z3 版本

- SMT 公式与问题定义等价：每个约束直接对应问题条件
- Z3 求解器是可满足性模理论（SMT）的决策程序，对一阶逻辑片段是完备的
- 因此：若 Z3 返回 SAT，则模型中的赋值即为合法布局；若 UNSAT，则问题确实无解

### 自定义算法

- 搜索是深度优先完备搜索，不遗漏任何可能的布局
- 所有剪枝都是充分条件：仅在"确定不可能"时触发，不会剪掉任何可行解
- 局部搜索不替代主搜索，不影响完备性
- 因此：若存在解，算法必能找到（完备性）；若返回 UNSAT，则确实无解（正确性）
