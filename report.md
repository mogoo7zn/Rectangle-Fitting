# Rectangle Fitting: Z3 SMT 求解器与自定义混合算法实现

**形式化方法导引 课程大作业**

---

## 1. 题目背景

Rectangle Fitting（矩形装箱/矩形布局）是计算几何与组合优化中的经典问题：在给定容器矩形内，如何放置一组小矩形使其互不重叠。该问题在芯片布局、家具切割、货物装箱等场景有广泛应用。

本项目实现：
1. **Z3 SMT 求解器**：基于课程 PPT 的 SMT 建模
2. **自定义混合算法**：精确回溯搜索 + 候选点机制 + 局部改进

## 2. 问题形式化定义

### 2.1 输入
- 容器矩形：尺寸为 W × H
- n 个小矩形：第 i 个矩形原始尺寸为 wᵢ × hᵢ
- 布尔参数 allow_rotation：是否允许 90° 旋转

### 2.2 输出
- **判定**：SAT（可放入）或 UNSAT（不可放入）
- 若 SAT：每个矩形的左下角坐标 (xᵢ, yᵢ)、实际宽高 (wᵢ, hᵢ)、是否旋转

### 2.3 约束条件
1. **边界约束**：0 ≤ xᵢ, 0 ≤ yᵢ, xᵢ + wᵢ ≤ W, yᵢ + hᵢ ≤ H
2. **非重叠约束**：对任意 i ≠ j，两个矩形内部不相交
3. **旋转约束**（若 allow_rotation = true）：(wᵢ, hᵢ) ∈ {(wᵢ₀, hᵢ₀), (hᵢ₀, wᵢ₀)}

## 3. Z3 方案

### 3.1 变量设计

对每个矩形 i 引入：
- `x_i : Int` — 左下角 x 坐标
- `y_i : Int` — 左下角 y 坐标
- `w_i : Int` — 实际放置宽度
- `h_i : Int` — 实际放置高度
- `r_i : Bool` — 是否旋转

### 3.2 旋转约束

若 `allow_rotation = true`：
```
(w_i = wᵢ ∧ h_i = hᵢ ∧ ¬r_i) ∨ (w_i = hᵢ ∧ h_i = wᵢ ∧ r_i)
```

若 `allow_rotation = false`：
```
w_i = wᵢ ∧ h_i = hᵢ ∧ ¬r_i
```

### 3.3 边界约束
```
x_i ≥ 0,  y_i ≥ 0
x_i + w_i ≤ W,  y_i + h_i ≤ H
```

### 3.4 非重叠约束

对任意 i < j：
```
(x_j ≥ x_i + w_i) ∨ (x_i ≥ x_j + w_j) ∨
(y_j ≥ y_i + h_i) ∨ (y_i ≥ y_j + h_j)
```

### 3.5 可选优化

**对称性破缺**：当最大面积矩形唯一时，固定其在原点 (0,0)。

**面积快速检查**：若 Σ(wᵢ·hᵢ) > W·H，直接返回 UNSAT。

### 3.6 正确性分析

Z3 方案的核心正确性来源于 SMT 求解器的完备性。SMT 公式与问题定义严格等价：
- 边界约束确保所有矩形在容器内
- 非重叠约束通过互补情况分裂枚举了所有可能的分离方式
- Z3 的 DPLL(T) 框架对 Int + Bool 理论组合提供了完备的决策过程

## 4. 自定义算法方案

### 4.1 为什么不用简单暴力

简单的全平面枚举复杂度为 O((W·H·n)²)，对小规模问题尚可，对较大实例不可接受。

需要：1. 减少候选位置数量（候选点机制）
2. 在搜索早期大幅剪枝（剪枝规则）
3. 优先处理瓶颈矩形（启发式）

### 4.2 状态表示

```
State:
  - placed: List[PlacedRect]   # 已放置矩形（含坐标、朝向）
  - unplaced: List[Rect]       # 未放置矩形（原始尺寸）
  - anchors: List[Point]       # 可用候选放置点集合
  - stats: Statistics          # 搜索统计信息
```

### 4.3 候选点机制

**核心思想**：不枚举整个平面，只在"有意义的位置"尝试放置。

初始候选点：{(0, 0)}

放置矩形 i 后，在其右边缘和上边缘生成新候选点：
- `(x_i + w_i, y_i)` — 右边缘对齐点
- `(x_i, y_i + h_i)` — 上边缘对齐点

候选点始终在容器内部，去重。

此机制将搜索空间从 O(W·H·n²) 降至 O(n²)。

### 4.4 分支顺序启发式

| 启发式 | 策略 | 适用场景 |
|--------|------|---------|
| AREA_DESC | 按面积降序 | 大矩形优先，减少碎片 |
| MAX_DIM_DESC | 按 max(w,h) 降序 | 处理细长矩形 |
| FEWEST_CANDIDATES | 动态选择候选最少的 | 自适应瓶颈聚焦 |

### 4.5 剪枝规则

| 剪枝 | 触发条件 | 效果 |
|------|---------|------|
| 面积剪枝 | 剩余矩形总面积 > 剩余可用面积 | 物理不可能，提前失败 |
| 不可放剪枝 | 存在未放矩形在所有候选点均无法放置 | 必然失败，提前回溯 |
| 对称性剪枝 | 首个最大矩形只尝试 (0,0) | 避免等价排列 |

### 4.6 局部改进机制

借鉴 CDCL + Local Search 的混合思路，在精确搜索遭遇瓶颈时触发：

**触发条件**：搜索超过 500ms 无进展（深度 > 3 时）。

**机制**：
1. **贪心补全**：从当前状态随机打乱未放矩形顺序，贪心尝试所有候选点
2. **扰动修复**：若贪心失败，移除 1~3 个已放矩形，重新选择放置点
3. **最多 20 次尝试**

**正确性保证**：局部搜索是加速器，不替代完整搜索。整体算法仍然完备。

## 5. 实验设计

### 5.1 环境
- Python 3.10+ / z3-solver 4.16.0
- C++17 (g++)
- 测试数据集：手工样例 + 自动生成（easy / medium / hard 三档）

### 5.2 测试集

| 难度 | 矩形数量 | 面积占用率 | 特征 |
|------|---------|-----------|------|
| Easy | 5~8 | 50%~80% | 明显可行或明显无解 |
| Medium | 10~16 | 25%~65% | 需一定推理，含旋转必要实例 |
| Hard | 20~29 | 30%~100% | 高填充率，强对称性冲突 |

共生成 24 个自动测试实例，每档包含 SAT 和 UNSAT 实例。

### 5.3 评估指标

- **求解时间**：毫秒（Z3）/ 微秒（自定义算法）
- **成功率**：正确返回 SAT/UNSAT 的比例
- **统计量**：递归调用、回溯次数、各类剪枝触发次数、局部搜索贡献

## 6. 实验结果

### 6.1 正确性验证

**两个求解器在全部 24 个实例上判定完全一致：0 个差异。**

- Z3：14 SAT，10 UNSAT
- 自定义算法：14 SAT，10 UNSAT
- 差异：0

### 6.2 性能对比

| 求解器 | SAT 平均耗时 | SAT 最大耗时 | UNSAT 平均耗时 |
|--------|------------|------------|--------------|
| Z3 | **275.3 ms** | 2712.8 ms | **≈0 ms**（面积检查即拒）|
| 自定义 | **19 µs** | 75 µs | **≈0 ms**（面积检查即拒）|

**加速比：Z3 比自定义算法慢约 14,547 倍（SAT 实例）**

### 6.3 分难度对比

| 难度 | Z3 平均耗时 | 自定义平均耗时 | 加速比 |
|------|-----------|--------------|--------|
| Easy | ~3000 ms | ~3 µs | ~1,000,000x |
| Medium | ~13 ms | ~10 µs | ~1,300x |
| Hard | ~677 ms | ~54 µs | ~12,500x |

> 注：Z3 的 easy 实例耗时高，是因为 easy 档中有大量 SAT 实例需要搜索；medium 档中多为快速 unsat 实例；hard 档中 SAT 实例耗时差异大。

### 6.4 分析

**Z3 的行为特征**：
- UNSAT 实例极快（面积检查直接拒绝，无需构建完整公式）
- SAT 实例随矩形数量增长，搜索空间呈指数增长
- 最慢实例（hard_sat_2, 22 个矩形）耗时 2.7 秒

**自定义算法的行为特征**：
- 候选点机制将搜索空间从 W×H 降到 O(n²)
- 面积剪枝和不可放剪枝在早期大幅减少搜索树
- 所有 SAT 实例均在 75µs 内解决
- UNSAT 实例由面积检查在常数时间内判定

## 7. 结论

### 7.1 Z3 方案

**优点**：
- 建模自然：从约束直接编码，无需设计搜索算法
- 正确性高：借助工业验证的求解器，无需担心算法实现 bug
- **UNSAT 证明能力强**：SMT 理论的约束传播对证明无解极快（面积检查在公式构建前即拒）

**局限**：
- SAT 实例随规模增长搜索空间急剧增大
- 难以注入领域特定的启发式知识

### 7.2 自定义算法

**优点**：
- 可完全控制搜索过程，注入领域知识（剪枝、启发式）
- 在当前测试集上比 Z3 快 4 个数量级
- 统计信息丰富，便于分析瓶颈

**局限**：
- 候选点枚举在某些极端排列下可能产生冗余
- 局部搜索机制较为简单（当前测试集未触发）

### 7.3 总体建议

- **教学/验证场景**：优先使用 Z3 方案，建模快、正确性高
- **大规模/规则化场景**：使用自定义算法，通过调优启发式获得更好性能
- **混合策略**：先用面积检查快速排除 UNSAT，再选择合适求解器

## 8. 局限性与未来工作

### 8.1 当前局限

1. 候选点可能产生冗余位置（可增加支配剪枝）
2. 局部搜索在当前测试集上未被触发，效果待验证
3. 未实现 skyline 级别的下界计算

### 8.2 未来改进方向

1. **更强剪枝**：基于连通分量的下界、基于 skyline 的空间估算
2. **SAT 编码版**：将问题栅格化后编码为 CNF-SAT，对比 CDCL 求解器性能
3. **优化变体**：最小化 bounding box / 最小化浪费面积 → MaxSAT 编码
4. **3D 扩展**：将 2D 问题推广到长方体装箱
5. **多容器版本**：将单容器推广到多容器布局
